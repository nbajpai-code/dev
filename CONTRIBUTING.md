## ğŸ”„ Pull Request Process

### Step 1: Fork the Repository
```bash
# Click "Fork" button on GitHub
# Clone your fork
git clone https://github.com/YOUR-USERNAME/dev.git
cd dev
```

### Step 2: Create a Branch
```bash
# Create a descriptive branch name
git checkout -b add-new-conference
# or
git checkout -b update-tool-pricing
# or
git checkout -b fix-broken-links
```

### Step 3: Make Your Changes
- Edit the relevant markdown files
- Follow existing formatting
- Keep changes focused and specific
- Test all links

### Step 4: Commit Your Changes
```bash
git add .
git commit -m "Add AWS re:Invent 2025 to SFO events"
# or
git commit -m "Update VS Code pricing information"
```

### Step 5: Push and Create Pull Request
```bash
git push origin your-branch-name
```
Then create a pull request on GitHub with:
- Clear title describing the change
- Description of what was added/updated
- Why the change is valuable
